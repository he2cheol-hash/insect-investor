# AI 구현 프롬프트 — 투자 멘탈 곤충 유형 테스트

> 이 문서를 AI 코딩 도구(Claude Code, Cursor, v0 등)에 **첨부한 구현 스펙(`투자_곤충유형_테스트_구현스펙.md`)과 함께** 넣으세요.
> 스펙 문서가 "무엇을 만들지", 이 문서가 "어떻게/어떤 순서로 만들지"입니다.

---

## 0. AI에게 주는 한 줄 지시 (복사해서 맨 앞에 붙이세요)

```
첨부한 "투자_곤충유형_테스트_구현스펙.md"를 그대로 구현해줘.
이 작업지시서의 빌드 순서와 검증 기준을 반드시 지켜.
각 단계가 끝나면 멈추고 결과를 보여준 뒤 다음 단계로 넘어가.
스펙에 없는 내용은 임의로 추가하지 말고, 모호하면 질문해.
```

---

## 1. 기술 제약 (반드시 준수 — 위반 시 처음부터 다시)

- **프레임워크**: Next.js (App Router) + TypeScript + Tailwind CSS
- **순수 CSR/SPA 금지** — SEO 때문에 결과 페이지는 반드시 SSG
- **금융 조언 금지** — 특정 종목·상품·매매시점·수익률 추천 일체 없음 (스펙 14장 표현 가이드 준수)
- **로그인·DB 없음** (MVP) — 테스트 상태는 클라이언트 메모리/URL로만
- **CLS 방지** — 모든 광고 슬롯은 로드 전에도 동일 높이 점유
- **모바일-퍼스트** — 트래픽 70%+가 모바일

## 2. 빌드 순서 (이 순서로, 단계마다 멈춰서 확인)

### STEP 1 — 프로젝트 초기화
- `create-next-app` (App Router, TypeScript, Tailwind, ESLint)
- 폴더 구조 제안:
  ```
  /app
    /test/page.tsx
    /result/[type]/page.tsx
    /articles/[slug]/page.tsx
    /privacy/page.tsx
    /terms/page.tsx
    page.tsx            (랜딩)
  /lib
    questions.ts        (17문항 데이터)
    types.ts            (8유형 데이터 + 스키마)
    scoring.ts          (채점 엔진)
    constants.ts        (축별 최대 원점수, fallback 좌표)
  /components
  ```
- **검증**: `npm run dev`로 빈 페이지가 뜨는지.

### STEP 2 — 데이터 구조화 (스펙 8·10·11장)
- `questions.ts`: 스펙 8장 Q1~Q17 전부. 각 선택지에 `{ label, axisDeltas: { 축: 점수 } }` 형태로 가중치 포함. **음수(Q8-Ⓑ 충동성 −1)까지 정확히.**
- `constants.ts`: 8.1 축별 최대 원점수 (impulse 16 / longTerm 15 / risk 12 / lossAversion 11 / crowd 10 / research 9), 9.5 fallback 좌표 JSON.
- `types.ts`: 스펙 10장 `InsectType` 스키마대로 8종 전부 (11장 텍스트 콘텐츠 포함).
- **검증**: 8유형 × (요약·강점·주의점·습관·관련콘텐츠) 누락 없는지, 17문항 가중치 합이 8.1 최대값과 일치하는지.

### STEP 3 — 채점 엔진 (스펙 9장 — 가장 중요)
순서대로:
1. **원점수 합산** (9.1)
2. **0~100 정규화** (9.2): `clamp(raw / maxRaw * 100, 0, 100)`
3. **dominant 판정 switch** (9.3) — 스펙의 의사코드 그대로
4. **동점 우선순위** (9.4): 충동성 > 손실회피 > 장기지향 > 정보탐색 > 위험선호 > 군중민감도
5. **유클리드 거리 fallback** (9.5) — 어떤 갈래에도 안 걸리면 가장 가까운 좌표
- **검증 (필수)**: 스펙 9.6의 검증 예시 3개를 단위 테스트로 작성하고 통과시킬 것.
  - 회피형 전부 선택 → 콩벌레
  - 회피형 + 충동성 높음 → 나비
  - Q8Ⓐ+Q13Ⓑ → 잠자리
- 임계값(50/55/40)은 상수로 빼서 나중에 튜닝 가능하게.

### STEP 4 — 화면 (스펙 4·5·16장)
- 랜딩 `/`: 서비스 소개, 시작 버튼, 8종 미리보기, 고지문
- `/test`: 17문항, 진행률 바, 뒤로가기, 답변 임시저장(클라이언트)
- `/result/[type]`: **SSG 8종**. 11장 구성 순서 그대로 (①곤충명 ②자학 한 줄 ③요약 ④강점 ⑤주의점 ⑥습관 ⑦희귀도 ⑧공유 ⑨관련콘텐츠 ⑩다시하기)
- 톤: 단정("맞다/틀리다") 금지, "그럴 수 있다/조심하면 좋다". 유형별 테마 색(6장) + 파스텔 배경.
- **검증**: 모바일 뷰에서 각 화면 깨짐 없는지, 8개 결과 URL이 직접 접근 가능한지.

### STEP 5 — 공유 (스펙 12장 — 공유 트래픽의 핵심, 절대 생략 금지)
- **동적 OG 이미지**: `@vercel/og`의 `ImageResponse`로 결과별 (곤충 + 유형명 + 자학 한 줄) 카드 생성
- 카카오 공유 SDK 또는 링크복사 + 이미지 저장
- 희귀도 %는 MVP에선 임시 추정치(상수)로 시작
- **검증**: 결과 URL을 OG 디버거(또는 카톡)에 넣었을 때 카드 이미지가 뜨는지.

### STEP 6 — 광고·분석·정책 (스펙 13·14·17장)
- AdSense 슬롯 자리 예약(높이 고정, 지연 로딩) — 결과 화면 ⑥~⑨ 사이
- GA4: `start_click`, `question_answered`, `result_generated`, `result_view`, `share_click`, `article_click`, `retake_click`
- **결과 분포 로깅** (8종이 고르게 나오는지)
- `/privacy`, `/terms` + 고지 문구(14.2) 푸터 노출
- sitemap.xml / robots.txt / 메타·OG·canonical 태그

## 4. AI가 자주 틀리는 함정 (미리 경고)

1. **정규화를 빼먹는다** → 충동성(최대16)이 항상 이김. 9.2 정규화 필수.
2. **음수 가중치 누락** → Q8-Ⓑ 충동성 −1을 빠뜨림. clamp 하한 0 처리도.
3. **결과 페이지를 CSR로 만든다** → SEO 망함. 반드시 SSG (`generateStaticParams`로 8종).
4. **OG 이미지를 생략** → 공유 트래픽의 핵심인데 "나중에"로 미룸. STEP5에서 반드시.
5. **금융 조언 표현이 새어 들어감** → 결과 텍스트에 종목/상품/타이밍 언급 금지.
6. **광고 슬롯 높이 미예약** → CLS 폭발. 로드 전에도 동일 높이.

## 5. 추천 진행 방식 (AI 도구별)

- **Claude Code / Cursor**: STEP 단위로 끊어서 지시 → 각 단계 결과 확인 후 다음. 한 번에 "전부 만들어"는 품질 떨어짐.
- **v0 / Bolt 등**: STEP 4(UI)부터 빠르게 프로토타입 → 채점 로직(STEP3)을 별도로 검증해 이식.
- 어느 도구든 **STEP3 채점 엔진은 9.6 테스트로 반드시 검증**하고 넘어갈 것.
