# 투자 멘탈 곤충 유형 테스트 — 구현 스펙 문서

버전: v2.0 (재기획)
목적: 이 문서 하나만으로 AI 또는 개발자가 전체 서비스를 바로 구현할 수 있도록 한 완전 명세
형식: 광고 수익형 모바일-퍼스트 반응형 웹 (성격유형 테스트)

---

## 0. 이 문서 사용법 (구현하는 AI/개발자에게)

이 문서는 "그대로 만들면 되는" 수준의 구현 스펙입니다. 읽는 순서와 우선순위는 다음과 같습니다.

1. 먼저 **1장(제품 의도)**과 **2장(수익 모델)**을 읽고 *왜* 이렇게 설계됐는지 파악할 것. 모든 UI/기술 결정은 "광고 수익 = 트래픽 × 체류 × 공유"라는 목표에서 역산된 것임.
2. **3장(기술 스택)**대로 프로젝트를 세팅. 핵심은 SEO를 위한 정적 생성(SSG)과 공유를 위한 동적 OG 이미지.
3. **6~10장**이 핵심 로직. 8개 유형, 6개 점수 축, 17문항(배점 포함), 채점 알고리즘이 전부 들어있음. 이 부분은 그대로 코드로 옮기면 됨.
4. **11장(결과 콘텐츠)**의 8종 텍스트를 데이터로 넣을 것.
5. 나머지 장(광고/SEO/정책/측정/로드맵)은 운영·확장 지침.

구현 우선순위: 테스트 진행 → 채점 → 결과 표시 → 공유 카드 → 광고/분석. 이 순서로 MVP를 완성.

핵심 제약 3가지 (절대 위반 금지):
- **금융 조언 금지**: 특정 종목·상품·매매시점·수익률 추천 절대 안 함. 재미/자기이해용 콘텐츠로만 포지셔닝.
- **Core Web Vitals 보호**: 광고가 LCP·CLS를 망치지 않도록 광고 슬롯 크기를 미리 예약.
- **개인정보 최소 수집**: 로그인 없음, 익명 사용, 개인식별 정보 미수집.

## 1. 제품 개요와 핵심 의도

### 1.1 한 줄 정의
사용자가 2~3분 안에 2~3지선다 문항에 답하면, 자신의 투자 행동 성향을 8종의 **곤충 캐릭터** 중 하나로 알려주고, 결과를 공유하며, 결과별 콘텐츠로 이어지는 모바일 웹 기반 재미형 투자 성향 테스트.

### 1.2 핵심 설계 철학 — "앞은 가볍게, 뒤는 깊게"
이 서비스의 가장 중요한 전략. 반드시 지킬 것.
- **앞(입구)**: 사용자에겐 가벼운 게임처럼 보여야 한다. 빠른 2~3지선다, 귀여운 곤충 캐릭터, 웃긴 자학 카피. 진입장벽 0, 설문조사 느낌 0.
- **뒤(엔진/확장)**: 그 뒤엔 6축 점수 체계와 결과별 교육 콘텐츠·도구가 깔려 있어, 한 번 들어온 사용자를 여러 페이지뷰로 전환한다. (광고 수익의 핵심)
- 즉 "심플하게"는 *기능을 빼는 것*이 아니라 *깊이를 뒤로 숨기는 것*이다.

### 1.3 왜 곤충인가
한국 개미 투자자 = 진짜 '개미(곤충)'. 투자자를 곤충으로 의인화하는 것이 주제와 완벽히 맞고, 캐릭터화가 공유(바이럴)를 폭발시킨다. 텍스트 유형명("뉴스 추격형")보다 캐릭터("불나방")가 결과 카드의 주인공이 되어 공유율을 높인다.

### 1.4 하는 것 / 하지 않는 것
- **하는 것**: 투자 행동 심리·습관·실수 패턴을 재미있게 보여주고, 결과별로 조심할 점과 공부 방향을 알려준다.
- **하지 않는 것**: 종목 추천, 금융상품 추천, 매매 타이밍 제안, 수익률 예측, 개인 맞춤 투자자문.

### 1.5 핵심 타깃
20~40대 직장인·사회초년생·투자 입문자, 주식/ETF/코인 관심자, 재테크 커뮤니티의 공유형 콘텐츠 소비자.

### 1.6 성공 기준(가설 검증 지표)
- 테스트 완료율 60% 이상
- 결과 공유율 8% 이상
- 결과 하단 재순환 클릭률 15% 이상
- 월 10만 PV 도달 가능성 검증

## 2. 수익 모델과 비즈니스 의도

### 2.1 이 서비스는 "광고 수익형 반응형 웹"이다
모든 설계의 최종 목표는 광고 수익. 광고 수익의 공식은 다음과 같고, UI/콘텐츠 결정은 전부 이 세 변수를 키우기 위한 것이다.

> 광고 수익 ≈ 트래픽(방문자) × 체류·페이지뷰(노출 기회) × 공유(추가 유입)

### 2.2 수익원
- **1차(MVP): 디스플레이 광고 (Google AdSense)** — 페이지뷰 기반. 트래픽이 쌓여야 의미가 생긴다. 한국어 콘텐츠 RPM은 낮은 편이라 월 수만~십만 PV가 목표선.
- **2차: 제휴 마케팅 (쿠팡 파트너스 등)** — 한국에선 같은 트래픽으로 디스플레이 광고보다 수익이 더 클 수 있음. 단, 금융 주제 특성상 투자 상품 제휴는 피하고 재테크 도서·일반 상품 등 정책 안전 범위에서만.
- **확장: 광고 제거 유료 옵션, 투자일지 템플릿, 프리미엄 리포트** (9장 참조).

### 2.3 왜 이 아이템이 광고에 유리한가
- **바이럴/공유 유입**: 결과를 캡처·공유 → 친구 유입. 느린 SEO에만 의존하지 않는다. 특히 한국은 유형 테스트 문화가 강함.
- **AI 검색에 강함**: 구글 AI 요약이 "테스트를 대신 풀어줄" 수 없다. 단순 정보글이 AI에 밀리는 시대에 살아남는 포맷.
- **체류·페이지뷰 구조**: 결과 로딩 → 긴 결과 해설 → 유형별 상세 → 관련 콘텐츠로 이어져 세션당 페이지뷰가 많다.

### 2.4 현실적 기대치 (구현자가 알아야 할 전제)
출시 후 2~3개월은 수익이 거의 0. 트래픽이 쌓이며 천천히 오른다. 따라서 초기엔 수익 최적화보다 **완료율·공유율·결과 분포 데이터 확보**가 우선.

## 3. 기술 스택과 아키텍처

### 3.1 권장 스택
- **프레임워크: Next.js (App Router)** — SEO를 위한 정적 생성(SSG)/서버 렌더(SSR)가 필수. 순수 CSR(SPA)은 색인 불리하므로 금지.
- **호스팅: Vercel** (무료 티어로 시작). 트래픽 확장 시 Cloudflare 등 고려.
- **도메인: 커스텀 도메인** (AdSense 승인에 사실상 필요).
- **스타일: 모바일-퍼스트 반응형**. 트래픽 70%+ 가 모바일. Tailwind CSS 권장.
- **상태: 테스트 진행 상태는 클라이언트 메모리/URL 상태로 충분.** 로그인·DB 없음(MVP).
- **분석: Google Analytics 4 + Google Search Console** 처음부터 연결.

### 3.2 핵심 기술 요구사항
- **유형별 결과 페이지는 정적 생성(SSG)** — `/result/[type]` 8개를 빌드 타임에 생성해 SEO 자산화.
- **동적 OG 이미지 생성** — 결과 공유 시 카톡·인스타에 곤충 캐릭터 결과 카드가 예쁘게 뜨도록. Next.js `ImageResponse`(@vercel/og) 사용. 이 한 가지가 공유 트래픽을 크게 좌우하므로 절대 빼지 말 것.
- **광고 슬롯 크기 사전 예약** — CLS 방지. 광고 영역은 로드 전에도 동일 높이를 점유.
- **사이트맵·robots.txt 자동 생성**, 모든 페이지에 적절한 meta/OG 태그.

### 3.3 성능 가드레일
- 광고가 LCP·CLS를 해치지 않게. 광고는 지연 로딩하되 슬롯은 예약.
- 곤충 캐릭터 이미지/SVG는 경량화. 결과 페이지 첫 화면(above the fold)은 광고보다 콘텐츠가 먼저.

## 6. 유형 체계 — 곤충 8종

각 유형은 6축에서 뚜렷하게 갈리도록 설계됨. 인접 유형끼리 "한 끗 차이"가 분명해야 사용자가 결과에 납득함(예: 사마귀=근거 서면 실행 / 잠자리=다 보고도 실행 못 함).

| # | 곤충(영문key) | 유형명 | 앵커 축 | 테마 색 | 자학 한 줄(공유 카피) |
|---|---|---|---|---|---|
| 1 | 불나방 (firefly) | 뉴스 추격형 | 군중민감도↑↑, 충동성↑↑ | coral `#993C1D` | "급등 뉴스 뜨면 손이 먼저 갑니다. 정신 차리면 이미 고점이에요." |
| 2 | 메뚜기 (grasshopper) | 한 방 점프형 | 위험선호↑↑, 충동성↑ | amber `#854F0B` | "한 방이면 인생역전! ...점프 잘못하면 어디 처박히긴 합니다." |
| 3 | 나비 (butterfly) | 기분 매매형 | 손실회피↑, 충동성 중(기분 변동) | pink `#993556` | "오늘 기분 따라 삽니다. 차트가 빨가면 제 멘탈도 빨개요." |
| 4 | 잠자리 (dragonfly) | 분석 과몰입형 | 정보탐색↑↑, 실행 지연(장기지향↓) | gray `#5F5E5A` | "눈이 360도라 다 보입니다. 다 보고, 또 보고... 그래서 아직 안 샀어요." |
| 5 | 사마귀 (mantis) | 조용한 저격수 | 정보탐색↑↑, 충동성↓↓ | purple `#534AB7` | "근거 없으면 손도 안 댑니다. 덕분에 빠른 기회는 자주 놓쳐요." |
| 6 | 거미 (spider) | 그물 분산형 | 장기지향↑, 정보탐색 중상, 충동성↓ | teal `#0F6E56` | "몰빵이요? 전 여기저기 그물 쳐두고 느긋하게 기다립니다." |
| 7 | 일개미 (ant) | 뚜벅뚜벅 적립형 | 장기지향↑↑, 충동성↓ | green `#3B6D11` | "화려하진 않은데요, 안 망해요. 매달 그냥 삽니다." |
| 8 | 콩벌레 (pillbug) | 현금 방어형 | 손실회피↑↑, 위험선호↓↓ | blue `#185FA5` | "위험하면 일단 몸을 맙니다. 제 돈은 예금에서 잘 자고 있어요." |

색 사용: 각 유형의 카드/결과 페이지 테마는 위 색 계열을 사용. 결과 카드는 해당 계열 파스텔 배경 + 같은 계열 진한 텍스트로 라이트/다크 모드 모두 자체 정합.

캐릭터 아트: MVP는 단순 SVG 글리프(또는 일러스트)로 시작 가능. 추후 전문 일러스트/생성 이미지로 업그레이드. 곤충은 모두 오리지널 디자인(저작권 안전).

## 7. 6축 점수 체계 (내부 엔진)

결과 8종은 *겉으로 보이는 캐릭터*, 6축은 *뒤에서 돌아가는 채점 기준*. 사용자에게 6축은 노출하지 않는다(원하면 결과 페이지에 레이더 차트로 부가 표시 가능).

| 축 (key) | 정의 | 해석 |
|---|---|---|
| 위험선호도 (risk) | 변동성·손실 가능성을 얼마나 감수하는가 | 높을수록 공격적 |
| 충동성 (impulse) | 뉴스·급등락에 즉각 반응하는가 | 높을수록 매매 기준이 흔들림 |
| 손실회피 (lossAversion) | 손실을 보는 것을 얼마나 못 견디는가 | 높을수록 손실 구간 스트레스 큼 |
| 정보탐색 (research) | 투자 전 자료를 얼마나 찾는가 | 높을수록 분석형, 과하면 결정 지연 |
| 장기지향 (longTerm) | 단기 등락 대신 장기 계획을 유지하는가 | 높을수록 적립·분산에 강함 |
| 군중민감도 (crowd) | 친구·커뮤니티·유튜브에 얼마나 휩쓸리는가 | 높을수록 FOMO 위험 |

## 8. 문항 17개 (전체, 배점 포함)

## 9. 채점 로직 (3단계)

### 9.1 1단계 — 원점수 합산
17문항 답변에서 선택지 가중치를 축마다 더한다. 결과: 6축 각각의 원점수(rawScore).

### 9.2 2단계 — 0~100 정규화
축마다 최대 원점수가 다르므로(8.1 표) 그대로 비교하면 충동성이 항상 이긴다. 따라서 정규화 필수.

```
normalized[axis] = clamp( rawScore[axis] / maxRaw[axis] * 100, 0, 100 )
```
clamp 하한 0 (Q8-Ⓑ의 충동성 −1로 음수가 나올 수 있어 0에서 끊음).

### 9.3 3단계 — 곤충 판정 ("최고 축 → 갈래" 방식)
정규화 점수에서 **가장 높은 축(dominant)** 을 찾고 보조 조건으로 분기.

```
dominant = argmax(normalized)   // 동점 시 9.4 우선순위

switch (dominant):
  case 군중민감도:        → 불나방(firefly)
  case 위험선호:          → 메뚜기(grasshopper)
  case 손실회피:
       if 충동성 >= 50    → 나비(butterfly)        // 기분 따라 사고팖
       else              → 콩벌레(pillbug)        // 돈을 안 굴림
  case 정보탐색:
       if 장기지향 >= 55  → 거미(spider)           // 분산·계획형
       elif 충동성 <= 40 and 장기지향 >= 40 → 사마귀(mantis)  // 근거 서면 실행
       else              → 잠자리(dragonfly)       // 분석 과몰입, 실행 못 함
  case 장기지향:
       if 정보탐색 >= 50  → 거미(spider)
       else              → 일개미(ant)
  case 충동성:           // 2순위 축으로 분기
       second = 2순위 높은 축(군중/위험/손실회피 중)
       if second == 군중민감도 → 불나방
       elif second == 위험선호 → 메뚜기
       elif second == 손실회피 → 나비
       else                   → 9.5 fallback
```

임계값(50, 55, 40 등)은 **시작값**이며 출시 후 실제 분포를 보고 튜닝(12장 참조).

### 9.4 동점 처리
dominant 축이 동점이면 다음 우선순위로 끊는다(해설이 더 또렷한 축 우선):
> 충동성 > 손실회피 > 장기지향 > 정보탐색 > 위험선호 > 군중민감도

### 9.5 Fallback — 거리 기반 (판정 불가 방지)
어떤 갈래에도 안 걸리거나 애매한 경우, 곤충마다 "이상적인 6축 좌표"를 정의하고 사용자 정규화 점수와의 유클리드 거리가 가장 짧은 곤충으로 보낸다. 이로써 "판정 불가"가 절대 발생하지 않는다.

프로토타입 좌표(축 순서: [위험선호, 충동성, 손실회피, 정보탐색, 장기지향, 군중민감도]):

```json
{
  "firefly":     [60, 85, 30, 25, 20, 90],
  "grasshopper": [90, 75, 20, 30, 25, 45],
  "butterfly":   [45, 65, 75, 30, 25, 55],
  "dragonfly":   [35, 35, 45, 85, 30, 35],
  "mantis":      [45, 20, 40, 85, 55, 25],
  "spider":      [45, 30, 40, 60, 80, 30],
  "ant":         [35, 20, 45, 40, 85, 25],
  "pillbug":     [15, 25, 85, 40, 45, 30]
}
```
이 좌표는 결과 페이지의 **레이더 차트**(선택)나 "당신과 가까운/먼 유형" 표시에도 재사용 가능.

### 9.6 검증 예시
- 손실 문항을 모두 회피형으로 선택(Q1Ⓐ·Q4Ⓐ·Q7Ⓐ·Q9Ⓐ·Q14Ⓐ): 손실회피 원점수 ~10~11 → 정규화 ~95, 위험선호 바닥, 충동성 낮음 → dominant=손실회피, 충동성<50 → **콩벌레**.
- 같은 회피 성향이지만 Q5Ⓑ(불타기)·Q13Ⓐ(일단 지름)으로 충동성도 높으면 → 손실회피 갈래에서 **나비**로 분기.
- Q8Ⓐ(한 번 더 확인)·Q13Ⓑ(재다 놓침): 정보탐색 dominant, 장기지향·실행 낮음 → **잠자리**.

## 11. 결과 페이지 콘텐츠 (8종 전체)

결과 페이지 구성 순서: ① 곤충 이름·캐릭터 ② 자학 한 줄(공유 카피) ③ 한 줄 요약 ④ 강점 ⑤ 주의점 ⑥ 필요한 습관 ⑦ 희귀도(%) ⑧ 공유 카드 버튼 ⑨ 관련 콘텐츠 ⑩ 다시하기. 광고는 ⑥과 ⑨ 사이 등 콘텐츠를 충분히 본 뒤 자연스러운 위치에.

**1. 불나방 — 뉴스 추격형**
- 요약: 정보를 빠르게 받아들이지만, 이미 오른 뒤 따라가기 쉽다.
- 강점: 시장 흐름·트렌드를 누구보다 빨리 캐치한다.
- 주의점: 고점 매수가 잦고, 뉴스·커뮤니티 분위기에 매매 기준이 흔들린다.
- 필요한 습관: 매수 전 "어제 알았어도 샀을까?" 자문 · 관심종목은 24시간 대기 후 결정 · 뉴스 보고 산 종목엔 손절선 필수.
- 관련 콘텐츠: FOMO 줄이는 법 / 뉴스 트레이딩의 함정.

**2. 메뚜기 — 한 방 점프형**
- 요약: 큰 수익 기회에 강하게 끌리고, 변동성도 함께 끌어안는다.
- 강점: 과감하고, 수익 잠재력이 큰 베팅을 두려워하지 않는다.
- 주의점: 한 번의 실패가 치명적일 수 있고, 비중 관리 없이 몰빵·레버리지로 가기 쉽다.
- 필요한 습관: 한 종목 최대 비중 룰 정하기 · 잃어도 되는 돈만 투입 · 불타기 전 익절 기준 먼저 세우기.
- 관련 콘텐츠: 비중 관리의 기술 / 레버리지의 진실.

**3. 나비 — 기분 매매형**
- 요약: 시장 변화에 민감하지만 감정이 매매를 좌우한다.
- 강점: 변화에 유연하게 반응한다.
- 주의점: 고점에 사고 저점에 파는 패턴, 일관된 기준 부재.
- 필요한 습관: 매매 일지로 감정 기록 · 충동 매수 전 하루 자기 · 자동 적립 등 규칙 기반 매매로 전환.
- 관련 콘텐츠: 감정 매매 끊는 법 / 투자 일지 쓰는 법.

**4. 잠자리 — 분석 과몰입형**
- 요약: 분석은 완벽한데 실행을 못 해 기회를 놓친다.
- 강점: 분석력·정보 수집이 뛰어나고 리스크를 미리 본다.
- 주의점: 완벽을 기다리다 실행 못 함, 안 사서 못 번 기회비용이 크다.
- 필요한 습관: "70% 확신이면 소액이라도 산다" 룰 · 분석 마감 시간 정하기 · 적립식으로 결정 자동화.
- 관련 콘텐츠: 결정장애 깨는 법 / 적립식 투자의 힘.

**5. 사마귀 — 조용한 저격수**
- 요약: 근거 기반의 절제된 매매, 한 번 정하면 흔들리지 않는다.
- 강점: 원칙이 있고 충동에 흔들리지 않는다.
- 주의점: 기준이 엄격해 빠른 기회를 놓치고, 변화한 시장에 늦게 반응할 수 있다.
- 필요한 습관: 기준 일부는 유연하게 · 소액 "탐색 포지션" 허용 · 정기 리밸런싱.
- 관련 콘텐츠: 원칙 투자 vs 유연성 / 리밸런싱 가이드.

**6. 거미 — 그물 분산형**
- 요약: 분산과 계획 유지에 강한 안정형.
- 강점: 비중 관리·분산이 탁월하고 계획을 지킨다.
- 주의점: 지나친 분산으로 수익이 밋밋해지거나 결정이 느릴 수 있다.
- 필요한 습관: 핵심-위성 전략 · 분산 개수 상한 설정 · 정기 점검.
- 관련 콘텐츠: 분산투자 제대로 하는 법 / 핵심-위성 전략.

**7. 일개미 — 뚜벅뚜벅 적립형**
- 요약: 꾸준함과 인내로 장기 복리를 누린다.
- 강점: 시장 등락에 안 흔들리고 적립을 지속한다.
- 주의점: 너무 보수적이라 기회를 놓치거나 점검 없이 방치할 수 있다.
- 필요한 습관: 연 1~2회 리밸런싱 · 적립액 정기 상향 · 비상금 분리.
- 관련 콘텐츠: 적립식 복리의 힘 / 리밸런싱 타이밍.

**8. 콩벌레 — 현금 방어형**
- 요약: 손실을 잘 안 보지만, 투자 시작 자체가 어렵다.
- 강점: 리스크 관리 본능이 강하고 위기에 안전하다.
- 주의점: 과도한 회피로 투자를 못 시작하고, 인플레이션에 자산가치가 깎인다.
- 필요한 습관: 아주 작은 금액부터 시작 · 안전자산 비중 정한 뒤 나머지 분산 · 손실은 장기 관점에서 보기.
- 관련 콘텐츠: 투자 첫걸음 가이드 / 예금만 하면 생기는 일.

---

## 12. 공유·바이럴 설계

- **동적 OG 이미지**(필수): 결과별 곤충 캐릭터 + 유형명 + 자학 한 줄을 담은 카드. 카톡·인스타·트위터 공유 시 자동 노출. 공유 트래픽의 핵심.
- **카카오 공유 SDK** 연동(또는 링크 복사 + 이미지 저장).
- **희귀도 %**: 출시 후 실제 응답 분포로 자동 계산해 "당신은 전체의 N%, 흔치 않은 OO형"으로 표시. 공유 욕구를 높임. (DB/카운터 필요; MVP는 임시 추정치로 시작 후 실데이터로 교체.)
- **투자 궁합(확장)**: "당신과 잘 맞는/상극인 곤충" 표시. 9.5 프로토타입 거리로 계산 가능. 친구 태그 유도에 강력.
- **공유 문구 자동 생성**: 자학 한 줄을 기본 공유 텍스트로.

## 13. 광고 배치 설계

- **AdSense 승인 전제**: 커스텀 도메인, 양질의 콘텐츠(결과 8종 + 교육 글 다수), 개인정보처리방침·이용약관 페이지, 정책 준수.
- 배치 원칙: 콘텐츠와 광고를 명확히 구분, 실수 클릭 유도 금지, 시작 버튼·핵심 CTA에 광고를 바짝 붙이지 않기.
- 질문 화면: MVP에선 중간 광고 지양(이탈 방지 우선). 굳이 넣는다면 8번 문항 뒤 네이티브 정도만.
- 결과 화면: 콘텐츠를 충분히 본 뒤 자연스러운 위치(습관/관련 콘텐츠 사이)에.
- **CLS 방지**: 모든 광고 슬롯은 로드 전 동일 높이 점유.
- 무효 트래픽·인위적 클릭 유도 금지(AdSense 정지 위험).

## 14. 법·정책·신뢰 가드레일

> 이 문서는 법률 자문이 아님. 실제 출시 전 금융 표현·광고 문구·개인정보처리방침·이용약관은 전문가 검토 권장.

### 14.1 넘지 말아야 할 선 (표현 가이드)
| 금지 영역 | 피할 표현 | 대체 표현 |
|---|---|---|
| 종목 추천 | "당신은 삼성전자/비트코인이 맞습니다" | "당신은 급등 자산에 흔들리기 쉬운 편입니다" |
| 상품 추천 | "레버리지 ETF가 맞습니다" | "변동성 큰 상품은 감정 관리가 중요합니다" |
| 매매 시점 | "지금 사야 합니다" | "매매 전 본인의 기준을 점검해보세요" |
| 수익 보장 | "이 유형은 돈을 잘 법니다" | "이 유형은 이런 실수를 줄이면 더 안정적일 수 있습니다" |
| 맞춤 자문 | "당신 자산에 맞는 포트폴리오" | "일반적인 자기점검 결과" |

### 14.2 고지 문구(모든 화면 또는 푸터/결과에 노출)
- 목적: 본 테스트는 재미와 자기이해를 위한 콘텐츠이며, 투자 조언 또는 투자권유가 아닙니다.
- 비추천: 특정 금융상품·종목·가상자산·매매 시점·수익률을 추천하지 않습니다.
- 책임: 투자 판단과 책임은 이용자 본인에게 있으며, 모든 투자는 손실 가능성이 있습니다.
- 개인정보: 로그인 없이 사용하며, 개인 식별 정보는 수집하지 않습니다.

### 14.3 참고 기준
AdSense 프로그램 정책·무효 트래픽 정책·광고 배치 정책 준수. 금융소비자보호법상 반복적·영리적 금융상품 자문처럼 보이지 않게 설계. 개인정보보호법상 MVP는 개인정보 수집 최소화.

## 15. SEO 설계

- 유형별 결과 페이지(`/result/[type]`) 8개를 SSG로 색인 자산화(예: "투자 성향 테스트 결과 - 불나방형").
- 결과별 교육 콘텐츠(`/articles/[slug]`)를 꾸준히 추가해 검색 유입 자산 축적. AI 검색에 안 밀리도록 단순 정보 나열보다 경험·비교·체크리스트형 콘텐츠.
- 모든 페이지: title/description/OG/canonical 메타, 구조화 데이터(가능 시), sitemap.xml 제출, robots.txt.
- 타깃 키워드 예: "투자 성향 테스트", "개미 투자자 유형", "내 투자 멘탈", "FOMO 테스트" 등.

## 16. 디자인·톤앤매너

- **모바일-퍼스트**, 카드 중심 UI. 결과 카드는 곤충 캐릭터가 주인공.
- 색: 6장 유형별 테마 색 사용. 파스텔 배경 + 동일 계열 진한 텍스트(라이트/다크 정합).
- 톤: 친근하지만 허세 없는. 투자 실패를 비난하지 않고 행동 패턴을 짚어줌. 결과명은 재미있게, 해설은 차분하게. 단정("맞다/틀리다") 금지, "그럴 수 있다/조심하면 좋다"로.
- 문항 선택지도 캐릭터 보이스로(예: Q1Ⓒ "물타기. 평단 낮출 기회잖아").

## 17. 측정·분석

- 이벤트: `start_click`, `question_answered`(문항번호 포함), `result_generated`(유형 포함), `result_view`, `share_click`, `article_click`, `retake_click`.
- **결과 분포 로깅(중요)**: 8종이 고르게 나오는지 추적. 특정 유형(예: 잠자리)이 너무 적게/많이 나오면 9.3 임계값을 조정. 초기 튜닝의 근거 데이터.
- 핵심 지표 대시보드: 완료율, 공유율, 재순환 클릭률, PV, RPM.

## 18. MVP 범위와 로드맵

MVP(4~6주): 테스트 1종(17문항), 결과 8종, 공유 OG 이미지, AdSense, GA4, 기본 SEO, 개인정보/약관 페이지.

이후 확장:
1. 결과별 교육 콘텐츠 글 다수(SEO 자산화)
2. 희귀도 %·투자 궁합 기능
3. 도구(수익률/복리 계산기 등) — 검색 유입 + 체류 강화
4. 광고 제거 유료 옵션, 투자일지 템플릿, 프리미엄 리포트
5. 추가 테스트 종류(다른 성격 테스트)로 교차 유입
